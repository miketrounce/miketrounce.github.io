How video works on a webpage

The video you just watched is a standard MP4 file sitting in the same folder as the HTML page. There is no streaming service or video platform involved.

The video tag

HTML has a built-in <video> element. You point it at a file, and the browser handles everything else — loading, buffering, play/pause controls, fullscreen, and volume. No JavaScript required.

How it loads

When you open the page, the browser sees the <video> tag and starts fetching the MP4 file from the server. With preload="metadata", it only downloads enough to show the duration and first frame until you press play. Then it streams the rest progressively.

Why not YouTube?

You could embed a YouTube or Vimeo player instead. The trade-off is control: a self-hosted file has no ads, no tracking, no external dependencies, and loads instantly from the same server as the page. The downside is file size — video files are large, and GitHub Pages has a 100MB per-file limit.
