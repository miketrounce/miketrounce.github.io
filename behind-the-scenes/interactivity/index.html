Behind the Scenes

How static pages can move

The games on Tricksy — Lula, Snake, Kaboom — are interactive. Numbers change, pieces move, the screen responds to your clicks. And yet these pages are written in plain HTML, CSS, and JavaScript. There is no framework, no build step, and no server running behind them. So how does it work?

JavaScript and the DOM

When your browser loads an HTML page, it builds an internal model of the page called the DOM (Document Object Model). JavaScript can read and change this model at any time. When it does, the browser instantly redraws the affected part of the screen.

Event listeners

JavaScript can listen for things that happen — a button click, a key press, a timer firing. When an event occurs, a function runs, updates the DOM, and the page changes. This is what makes a game tick: the player clicks, JavaScript processes the move, updates the numbers, and the browser repaints.

CSS transitions

Some of the visual movement — fading, sliding, colour changes — is handled entirely by CSS. You define a start state and an end state, and the browser smoothly animates between them. No JavaScript needed for the animation itself, only a trigger to change a class or property.

No magic, just speed

Modern browsers can repaint the screen sixty times per second. A few lines of JavaScript changing a number and updating the text on screen happens so fast it feels like a living application. But underneath, it is still just an HTML file.
